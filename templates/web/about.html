About Us

As for this ConcordiaDoor project is to provide a platform for students who come from different programming backgrounds in the course Software Engineering.

The SE course aims to teach students how to design and implement complex software from the very beginning, but it does not limit the coding language. Thus, when students group up with different coding languages, it does take amount of time for them to negotiate with each other and may out come with disagreements, also, some of the students might need to discard the language that they are familiar with in this situation. Although students have programming knowledge, they still have the problem with: -Lack of experience in software specification and design. -Lack of experience in web development. -Lack of experience in medium/large group of software development

To avoid this kind of trouble, the ConcordiaDoor will provide an example application using best practices that students can use to learn or polish their web development skills. This web application project will use Python via Django framework, which both are easy even for new hand users to learn.

ConcordiaDoor is also a project to support students in acquiring design and development skills. It will support laboratory and homework that enable the students to gradually learn the skills, by extending the platform and to have an initial example on which to build on. In this first phase of ConcoordiaDoor, we focus on the customer application, other functions will be set as 'extension points'.

In this project, we use the UML diagrams at beginning to document the process of one complete software development circle, then C4 diagrams are also designed and shown for non-technical stuff to understand one project.
